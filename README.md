# epark_docs

EPARK の公開用ドキュメント、キャンペーンページ、特設ページ、ガイドページを管理する静的コンテンツ用リポジトリです。

## 概要

このリポジトリには、主に `epark.jp/docs/` 配下で公開される HTML ベースのページが含まれています。

- キャンペーン、特集ページ
- EC クーポン、ギフト関連ページ
- サービス紹介、ポイント案内
- ポリシー、ヘルプ、各種案内ページ
- 一部 PHP 断片ファイル（例: `feature.php`）

## ディレクトリ構成

主要なディレクトリは以下の通りです。

- `campaign/` : 年度別キャンペーン、特集ページ
- `ec/` : EC・クーポン・ギフト関連ページ
- `gps/` : ジャンル別導線ページ
- `service/` : サービス紹介ページ
- `news/` : お知らせページと関連データ
- `policy/` : ポリシー関連ページ
- `help/` : ヘルプ関連ページ
- `point-get/` : ポイント案内ページ
- `relaxation-esthe/` : リラク・エステ関連特設ページ
- `pet/` : ペット関連ページ
- `specified_commercial_transaction_law/` : 特定商取引法表記ページ
- `40x/` : エラーページ系コンテンツ

## ローカル確認

ルート全体には共通のビルド処理はありません。基本的には静的 HTML をそのまま確認します。

簡易確認例:

```bash
cd /Users/owner/サーバーから/epark_docs
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000/` を開き、対象ページへアクセスしてください。

例:

- `http://localhost:8000/campaign/`
- `http://localhost:8000/ec/`
- `http://localhost:8000/service/web/`

## 補足

- 一部ページは `app/` と `web/` で出し分けされています。
- 一部コンテンツは本番向けの絶対パスや外部アセット参照を含むため、ローカル表示が本番と完全一致しない場合があります。
- `campaign/comeback2025/` には `package.json` があり、個別に Tailwind CSS を利用する構成になっています。

## 作業時の目安

- 既存ページは静的 HTML を直接編集するケースが中心です。
- 画像、OGP、canonical、外部リンク、キャンペーン期間の記載ゆれに注意してください。
- `app/` と `web/` の両方があるページは、必要に応じて差分漏れがないか確認してください。
